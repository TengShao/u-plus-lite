# U-Minus 实现计划

## Goal
Build "U-Minus" (结了吗你), a team monthly workload tracking web app. Designers log man-days per requirement group per billing cycle. Admins manage cycles, complete requirements, and assign member levels. All computed metrics (换算人天, 投入比, 健康度, 推荐评级) are calculated at API response time.

## Architecture
- **Full-stack**: Next.js 14+ App Router (React + Tailwind CSS)
- **Backend**: Next.js Route Handlers (`/api/*`)
- **ORM**: Prisma with SQLite (WAL mode)
- **Auth**: NextAuth.js (Credentials Provider, JWT strategy)
- **Concurrency**: Optimistic locking (version field) for shared fields; per-user workload isolation via unique constraint

## Tech Stack
- next@14, react@18, tailwind css@3, prisma@5, next-auth@4, bcryptjs, typescript

---

### Task 2: Prisma Schema & Database

**Files**: `prisma/schema.prisma`, `src/lib/prisma.ts`

**Steps**:
1. Define schema with models: User, BillingCycle, RequirementGroup, Workload
   - User: id, name (unique), password, role (ADMIN/MEMBER), level (P5/P4/P3/INTERN/OUTSOURCE, optional)
   - BillingCycle: id, label, startDate, endDate, status (OPEN/CLOSED), createdBy → User
   - RequirementGroup: id, name, status (INCOMPLETE/COMPLETE), rating, module, pipeline, types (String/JSON), budgetItem, canClose, isBuilt, funcPoints, pageCount, version (default 1), createdInCycleId → BillingCycle, createdBy → User, lastSubmittedAt
   - Workload: id, userId → User, requirementGroupId → RequirementGroup, billingCycleId → BillingCycle, manDays (Float), unique constraint on [userId, requirementGroupId, billingCycleId]
2. Create Prisma client singleton (`src/lib/prisma.ts`)
3. Run `npx prisma migrate dev --name init`
4. Verify: `npx prisma studio` opens and shows empty tables

---

### Task 3: Constants & Computed Field Helpers

**Files**: `src/lib/constants.ts`, `src/lib/compute.ts`, `src/types/index.ts`

**Steps**:
1. Define TypeScript types and next-auth module augmentation (add userId, role, level to session)
2. Define constants:
   - `LEVEL_COEFFICIENTS`: { P3: 1, P4: 1.3, P5: 2, OUTSOURCE: 1, INTERN: 0.2 }
   - `RATING_STANDARDS`: { S: 20, A: 10, B: 5, C: 2, D: 0.5 }
   - `BUDGET_ITEMS`: Record<Pipeline, string[]> — all 80+ budget items grouped by pipeline
3. Define compute helpers:
   - `getConvertedManDays(manDays, level)` → manDays × coefficient
   - `getRecommendedRating(totalConvertedManDays)` → S/A/B/C/D
   - `getHealthStatus(ratio)` → 适合/过饱和/欠饱和
   - `getRatingStandard(rating)` → number
   - `getInputRatio(totalConverted, ratingStandard)` → percentage
4. Verify: unit logic is correct by manual inspection

### Task 7: Cycles API

**Files**: `src/app/api/cycles/route.ts`, `src/app/api/cycles/[id]/route.ts`

**Steps**:
1. GET `/api/cycles`: return all cycles ordered by startDate desc, grouped by year
2. POST `/api/cycles`: admin-only, auto-calculate label/startDate/endDate based on current date (26th rule), create cycle with status=OPEN
3. PATCH `/api/cycles/:id`: admin-only, toggle status OPEN↔CLOSED
4. Verify: CRUD operations work correctly

---

### Task 8: Requirements API

**Files**: `src/app/api/cycles/[cycleId]/requirements/route.ts`, `src/app/api/requirements/route.ts`, `src/app/api/requirements/[id]/route.ts`, `src/app/api/requirements/[id]/complete/route.ts`

**Steps**:
1. GET `/api/cycles/:cycleId/requirements`: return requirement groups visible in this cycle (have workload in cycle OR status=INCOMPLETE and created before/in cycle), include all workloads with user info, compute all dynamic fields (总投入人天, 总换算人天, 投入比, 健康度, 推荐评级, 功能点数推荐值), support filter params (pipeline, rating, health, designer, canClose, q), sort: INCOMPLETE first then COMPLETE
2. POST `/api/requirements`: create new requirement group with createdInCycleId and createdBy
3. PATCH `/api/requirements/:id`: update shared fields with optimistic lock (check version match, increment version), enforce: members can only edit when cycle is OPEN
4. DELETE `/api/requirements/:id`: delete requirement group and associated workloads, enforce: members only when cycle OPEN
5. PATCH `/api/requirements/:id/complete`: admin-only, set status=COMPLETE
6. Verify: all endpoints return correct data with computed fields
